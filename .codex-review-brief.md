## What I changed (commit 2d863fb, "r130.5")
1. post-install/33-network.sh — fix wired NIC never getting DHCP. Root cause I
   found: d-i netcfg writes /etc/network/interfaces with the install NIC as
   `allow-hotplug <if>` + `iface <if> inet dhcp`, and stock NetworkManager.conf
   ships `[ifupdown] managed=false`, so NM marks that NIC unmanaged (reason 76)
   and hands it to ifupdown, which never raises it on the XFCE/NM desktop. My fix:
   flip `[ifupdown] managed=true` and overwrite /etc/network/interfaces to
   loopback-only so NM owns all NICs. (Verified live on the OLD install: enp1s0
   immediately connected + got DHCP.)
2. post-install/60-plymouth.sh — switch from a never-applied bespoke 'nclawzero'
   script theme to the STOCK Ubuntu 'spinner' theme rebranded with the NCZ logo:
   crop NCZ lockup into spinner/watermark.png, tint BackgroundStart/EndColor to
   0x0b0f14, pin spinner via update-alternatives --set, set plymouthd.conf
   Theme=spinner, rebuild target-kernel initramfs. NOTE: this hook ALSO runs
   `systemctl mask plymouth-quit.service plymouth-quit-wait.service` (pre-existing
   line, r51). In OLD builds the plymouth packages were missing from the offline
   pool, so the hook hit `|| exit 0` at the apt-get line and NEVER reached the
   mask. In r130.5 the packages ARE bundled, so the hook now runs fully and the
   mask is applied for the FIRST time.

  in 60-plymouth.sh, now active for the first time. Masking plymouth-quit /
