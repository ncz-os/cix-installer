# 2026-05-07 — Cross-platform embedding throughput (MNEMOS corpus)

Same MNEMOS corpus across all engines: 8038 records, ~14 MB content text, mean 1763 chars, p95 2630, max 82K. Same `nomic-embed-text-v1.5.Q8_0` GGUF model on every CPU/GPU path; `bge-small-zh-v1.5_256.cix` Zhouyi-quantized model on the NPU path.

## Framing — application fit, not winners

Each platform is engineered for a different job. This page reports throughput **alongside** thermal envelope, hardware cost, and the workload it suits. None of these numbers should be read as "platform A beats platform B." The right read is:

> *"This is what each platform looks like at this specific workload (sentence embedding, ~256-token average, INT8/Q8 quantized) on this specific corpus. Pick the platform whose envelope matches your job."*

## Engines benched

### CPU path (apples-to-apples — same script, same model, same corpus)

| Box / silicon | rec/sec | p50 | wall (s) | embed_dim | notes |
|---|---|---|---|---|---|
| NCZ Magnetar — Cix Sky1 12-core ARM CPU | **2.97** | 436 ms | 2702 | 768 | nomic-q8 GGUF, llama-cpp-python in-process, n_threads=12 |
| PYTHIA — Intel Core 5 210H | **1.94 sustained** (in-flight) | TBD | TBD | 768 | identical script + model + corpus |

Same code path, same model — within ~5%, the ARM and x86 mini-PCs draw similar power for this CPU-only workload. The ratio is throughput-per-dollar at peer cost class. **Neither box is engineered to do embeddings on the CPU**; this is the floor, not the ceiling, for either platform.

### NPU path (Cix only — Cix is the only platform tested with a published NPU runtime today)

| Box / silicon | rec/sec | p50 | wall (s) | embed_dim | notes |
|---|---|---|---|---|---|
| NCZ Magnetar — Cix Zhouyi V3 NPU | **54.86** | 14.6 ms | 146 | 512 | bge-small-zh-v1.5 INT8 .cix, cix-noe-umd 2.0.2 |

This is what the Cix NPU is **for** — INT8 quantized encoder workloads. The peer test "PYTHIA via Intel iGPU + OpenVINO" is a fair comparison for the iGPU path and **is not in this run** (future bench, see *what's missing* below). PYTHIA does not ship a discrete NPU; the 210H predates Intel's NPU SKUs (Lunar Lake / Arrow Lake H bring NPU into the NUC class — we'll bench those when one is in the fleet).

### Discrete-GPU path (different tier — context, not peer)

| Box / silicon | rec/sec | p50 | wall (s) | TDP class | notes |
|---|---|---|---|---|---|
| TYPHON — RTX 5060 (consumer 8 GB GPU, Blackwell GB206, sm_120) | **166.30** | 7.1 ms | 48.3 | ~115 W (GPU) | nomic-q8 GGUF, llama-cpp-python built from source `-DGGML_CUDA=on`, `n_gpu_layers=99` |
| Brev cloud — ARM + GH200 / GB200 (proposed) | TBD | TBD | TBD | 400–700 W | rental at $0.5–5 / GPU-hr; relevant for batch one-shot indexing |

These belong in the table for context — the kit must run on them — but they are **not peer comparisons** to the $700 mini-PC class. A 115 W GPU plus its host is not a $700 fanless box; throughput alone is misleading without the envelope.

### Per-watt and per-dollar — different envelopes show

The same numbers, normalized:

| Engine | rec/sec | rough TDP of accelerator | rec/s per watt | hardware $ | rec/s per $1000 |
|---|---|---|---|---|---|
| Cix Sky1 NPU | 54.86 | ~2 W (NPU only) | **~27** | $700 box | 78 |
| Cix Sky1 12c ARM CPU | 2.97 | ~30 W (full SoC) | 0.10 | $700 box | 4 |
| PYTHIA Intel 210H CPU (in-flight) | 1.94 sustained | ~50 W (avg) | 0.04 | $800 box | 2 |
| TYPHON RTX 5060 | 166.30 | ~115 W (GPU) | 1.45 | ~$1800 box | 92 |

The story:
- **RTX 5060** wins absolute throughput at this workload — 3× the Cix NPU per record.
- **Cix Sky1 NPU** wins per-watt by ~19× — and that ratio is what lets it run 24/7 on a fanless edge appliance.
- **Per-dollar throughput is roughly tied** between Cix NPU and RTX 5060 at this workload, but the *deployment shape* is completely different: a fanless 1L box drawing the power of a phone charger vs a workstation drawing the power of an incandescent bulb.

Neither is "better." They are engineered for different deployment shapes.

## Application fit

| Workload | Best fit | Why |
|---|---|---|
| Always-on agent memory, edge, 24/7 | **Cix Sky1 NPU** | $700 box, fanless, ~2 W on the NPU itself, 55 rec/s sustained. No other platform tested matches the always-on cost envelope. |
| Mini-PC desktop / homelab without NPU | **PYTHIA-class Intel CPU + iGPU via OpenVINO** | Intel iGPU on a NUC15 has real OpenVINO embedding throughput (untested here; community benchmark welcome). Comparable price + form factor to Cix. |
| Dev workstation that already has a GPU | **CUDA / ROCm consumer GPU** | If the GPU is bought for graphics or LLM work anyway, embeddings ride free on it. TYPHON RTX 5060 is the data point. |
| Batch one-shot indexing of 10M+ docs | **Cloud GPU rental** (NVIDIA / AMD) | Spin up, index, tear down. Not always-on; not the right tool for a 24/7 memory layer. |
| Embedded / SBC with no NPU | **CPU baseline (this bench)** | Pi class, Allwinner, Rockchip without NPU usage. Numbers in this table set the ceiling. |
| Apple ecosystem | **MLX on M-series** | Future bench — Mac Studio in the fleet. |

The kit picks the right engine for the host. None of these is "best"; they are different envelopes.

## Methodology

- Bench script: `cix_inprocess_bench.py` (CPU + GPU paths via env vars) and `cix_npu_bench_v3.py` (NPU path).
- All paths use the same MNEMOS corpus pulled from PYTHIA via `/v1/export`.
- Embeddings computed in-process — no HTTP RPC, no llama-server overhead.
- NPU path: `from utils.NOE_Engine import EngineInfer` (cix-noe-umd 2.0.2 + libnoe 2.0.0, FyrbyAdditive prebuilt KMD).
- CPU + CUDA paths: `from llama_cpp import Llama` with `n_gpu_layers=0` (CPU) or `n_gpu_layers=99` (full GPU offload), `embedding=True`.
- TDP figures are platform spec sheets, not measured during the bench. Real-world embedding-only power is below spec TDP for CPU paths and at-spec for GPU paths.

## Headline

There is no winning silicon. There is silicon that fits the job. **Cix Sky1 NPU is engineered for the always-on edge memory layer** and dominates that envelope. Intel CPU + iGPU and AMD CPU + iGPU dominate the mini-PC desktop envelope. NVIDIA / AMD discrete GPUs dominate the batch-rental and dev-workstation envelopes. Apple MLX dominates the creative-workstation envelope. The kit lets the user write the same code once and run it on any of them — and lets the user pick by what they're paying per watt, per dollar, per hour, per use case.
