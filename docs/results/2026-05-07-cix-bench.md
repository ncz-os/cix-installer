# 2026-05-07 — Cix Sky1 + PYTHIA embedding throughput

Apples-to-apples (where possible) on the **same MNEMOS corpus** (8038 records, ~14 MB content text, mean 1763 chars, p95 2630, max 82K).

## In-process llama-cpp-python — `nomic-embed-text-v1.5.Q8_0` GGUF (768-dim)

Same model, same script (`cix_inprocess_bench.py`), same corpus, same `n_threads=12 n_ctx=8192`. Difference is silicon only.

| Host | Silicon | rec/sec | p50 | p95 | wall (s) | embed_dim |
|---|---|---|---|---|---|---|
| ncz-2533e4c9 (.66) | Cix Sky1 ARM64 12-core CPU (~30W) | **2.97** | 436 ms | 457 ms | 2702 | 768 |
| pythia (192.168.207.67) | x86_64 12-core (running, in-flight) | **1.94** sustained | TBD | TBD | TBD (eta ~32 min from 17:32 UTC) | 768 |

Cix CPU is ~1.5x PYTHIA CPU on this workload despite the ~3x lower TDP. ARM-NEON path in llama.cpp is well-tuned for nomic-q8.

## NPU — `bge-small-zh-v1.5_256.cix` (Zhouyi V3 INT8 quantized, 512-dim, 256-token max)

Different model (NPU model is encoder-only, INT8, fixed-shape) but same corpus and same MNEMOS task.

| Host | Silicon | rec/sec | p50 | p95 | wall (s) | embed_dim |
|---|---|---|---|---|---|---|
| ncz-2533e4c9 (.66) | Cix Zhouyi V3 NPU (~2W) | **54.86** | 14.6 ms | 18.5 ms | 146 | 512 |

NPU is **~18x Cix CPU**, **~28x PYTHIA CPU** on the same corpus.

## Methodology

- Bench script: `cix_inprocess_bench.py` (Cix CPU + PYTHIA paths) and `cix_npu_bench_v3.py` (NPU path).
- Both use the same MNEMOS corpus pulled from PYTHIA via `/v1/export`.
- Embeddings computed in-process — no HTTP RPC, no Ollama/llama-server overhead.
- NPU path goes through `from utils.NOE_Engine import EngineInfer` (cix-noe-umd 2.0.2 + libnoe 2.0.0, FyrbyAdditive prebuilt KMD).
- CPU path goes through `from llama_cpp import Llama` with `n_gpu_layers=0`, `embedding=True`.

## Headline

A 30W ARM SoC with a dedicated 30 TOPS NPU embeds 8000 MNEMOS records in **2 minutes** vs the 12-core x86 workstation's **40+ minutes** on the same corpus. **Different silicon, different purpose, different scale.** This is what NPUs are for.
