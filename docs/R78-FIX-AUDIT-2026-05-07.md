## Findings + resolution status

### HIGH - early_command heredoc is not a valid embedded preseed command

> File: preseed/preseed-ubuntu.cfg:103-145
>
> preseed/early_command is a debconf string. The heredoc body must be
> part of that same value. Here the preseed continuation stops at
> line 114:
>
>     cat > /tmp/sshd-watcher.sh <<'EOWATCHER'
>
> Lines 115-142 are naked shell-script lines in the preseed file, not
> part of early_command. Lines 143-145, including chmod and nohup
> launch, are also outside the value.
>
> Expected outcomes: strict parser rejects/misparses the file, or
> tolerant parser ignores the naked lines and executes an incomplete
> heredoc opener. Either way, the watcher is not reliably created or
> launched.

**STATUS: FIXED in commit `1c6153c`.** Watcher externalized to
`preseed/sshd-watcher.sh`, staged into the ISO at
`/cixmini/sshd-watcher.sh` by `build/build-iso-di.sh:1063`. early_command
now does:
```
cp /cdrom/cixmini/sshd-watcher.sh /tmp/sshd-watcher.sh ; \
chmod +x /tmp/sshd-watcher.sh ; \
/tmp/sshd-watcher.sh </dev/null >/dev/null 2>&1 & \
true
```
Every line in early_command now ends in `\`, so d-i's preseed parser
collapses to a single shell command correctly. Independent finding +
fix in parent session matched Codex's recommendation.
